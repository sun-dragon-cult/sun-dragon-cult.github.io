:::warning

This is a major release that requires Foundry v14. It introduces new V2 application sheets, migrates
to Foundry DataModels, and includes breaking changes to Active Effects. Make sure to back up your
world before updating.

:::

This release is a large overhaul of the system internals. The actor and item sheets have been
rewritten to use Foundry's V2 application framework, the data layer has been migrated to DataModels,
and Active Effects have been reworked to use the new field operation system. There are also many bug
fixes and quality of life improvements.

## V2 Actor & Item Sheets

<GithubIssue issue="845" repo="fvtt-system-rqg" />
<GithubIssue issue="852" repo="fvtt-system-rqg" />

The actor and item sheets have been completely rewritten using Foundry's Application V2 framework.
This brings better performance, a more modern look, and a more maintainable codebase. The V2 sheets
are now the default for all actors and items.

The new sheets also use native AppV2 tabs and improved drag-and-drop handling.

## Migration to DataModels

<GithubIssue issue="855" repo="fvtt-system-rqg" />

All system data has been migrated to use Foundry DataModels. This provides strict typing and
validation of all system data, which helps prevent data corruption and makes the system more robust.
The migration will run automatically when you first load a world with this version.

## Active Effects Overhaul

<GithubIssue issue="919" repo="fvtt-system-rqg" />

:::caution

This is a breaking change. Active Effects have been migrated to use DataModel field operations
instead of the legacy key-value approach. Existing Active Effects will be automatically migrated,
but custom Active Effects may need to be reviewed.

:::

Active Effects now use the `system.effect.add` namespace for additive effects. The system also
includes a centralized AE rewrite engine with rule-based processing and summary reporting to help
you understand what was migrated.

Additionally, Active Effects now sync their suspension state with the equipped status of the item
they belong to, and new effects use the world default settings.

## Structured Migration Reporting

<GithubIssue issue="891" repo="fvtt-system-rqg" />
<GithubIssue issue="902" repo="fvtt-system-rqg" />

When the system migrates your world data, it now provides a detailed migration report showing
exactly what was changed. This includes change diffs and migration metadata so you can review what
happened. There is also a recovery UI for save failures.

## CHA Rune Magic Point Limit Per Cult

<GithubIssue issue="839" repo="fvtt-system-rqg" />

The actor sheet now shows the CHA-based rune magic point limit for each cult, making it easier to
see how many rune points a character can have in each cult.

## Weapon Effect Modifiers in Combat

<GithubIssue issue="918" repo="fvtt-system-rqg" />

Weapon effects that modify attack or defence chance are now shown in the combat UI. This makes it
clearer what bonuses or penalties are being applied during combat rolls.

## Rune Display by RQID

Runes are now paired and displayed by their RQID, providing a more consistent and organized
presentation of rune information on the character sheet.

## Don't Hide Range & Duration for Ritual Rune Magic

<GithubIssue issue="848" repo="fvtt-system-rqg" />

Ritual Rune Magic spells now show their Range and Duration fields instead of hiding them, since
these values can still be relevant for ritual casting.

## Highest-Priority RQID Document Resolution

<GithubIssue issue="838" repo="fvtt-system-rqg" />

When resolving RQID links, the system now returns the highest-priority document regardless of
whether it comes from the world or a compendium. This means world-level overrides are properly
respected.

## Protection Token Effects Support All Creature Types

<GithubIssue issue="837" repo="fvtt-system-rqg" />

Protection token effects now use regex RQIDs, allowing them to work with all creature types instead
of only the default hit locations.

## Derived Calculations in DataModel

<GithubIssue issue="880" repo="fvtt-system-rqg" />

Derived calculations (like skill category modifiers and hit point totals) have been moved into the
DataModel layer. This enables Active Effect-driven derived updates, making the data flow more
predictable and correct.

## End-to-End Testing

The system now includes Playwright-based end-to-end tests that verify core functionality against a
running Foundry instance. This helps catch regressions early and ensures the system works correctly
across updates.
